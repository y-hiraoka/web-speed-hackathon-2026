# 画像アセットが未最適化（高解像度JPG、レスポンシブ対応なし）

## カテゴリ

アセット

## 影響するメトリクス

LCP / TTFB

## 影響度

高

## 作業規模

M

## 問題の原因

`public/images/` 内の30枚の画像が以下の問題を持つ：

1. **過大な解像度**: 3549x5323, 6531x4354, 5304x6630 など、Web表示に不要な超高解像度
2. **大きなファイルサイズ**: 1枚あたり 2-7MB、合計約 80MB
3. **JPG のみ**: WebP や AVIF などの効率的なフォーマットを使用していない
4. **レスポンシブ画像なし**: `srcset` / `<picture>` が使われておらず、モバイルでも高解像度画像がそのまま配信される
5. **遅延読み込みなし**: `loading="lazy"` が設定されていない

プロフィール画像（30枚、合計3.2MB）も同様に最適化されていない。

## 原因箇所

`application/public/images/` — 30枚のJPG画像（合計約80MB）
`application/public/profile-images/` — 30枚のプロフィール画像（合計約3.2MB）
`application/client/src/utils/get_path.ts` — パス生成（サイズバリエーション非対応）

## 解決方法

1. 画像を適切なサイズにリサイズ（表示サイズの2倍程度）
2. WebP または AVIF フォーマットに変換
3. 品質を 80-85% に設定
4. 必要に応じて複数サイズのバリエーションを生成し `srcset` で提供
5. `loading="lazy"` を追加（ファーストビュー以外の画像）
6. シードスクリプトで画像変換を行うか、ビルド時に処理

## 依存関係・注意点

- VRT でデザイン差異が検出されないよう、品質を下げすぎないこと
- プロフィール画像は小さく表示されるため、大幅にリサイズ可能
